# Implementation Plan: Recipe Image Upload

**Branch**: `002-recipe-image` | **Date**: 2026-03-08 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-recipe-image/spec.md`

## Summary

Add recipe cover image upload with client-side compression. Cover images display as backgrounds on recipe cards (with semi-transparent overlay for readability) and as hero images on recipe detail pages. Users can upload from media library or camera, replace existing images, and remove them. Shared recipe recipients can view but not modify images.

## Technical Context

**Language/Version**: TypeScript 5.x / Node.js 20+
**Primary Dependencies**: Next.js 15 (App Router), @supabase/supabase-js, @supabase/ssr, Tailwind CSS v4, shadcn/ui, browser-image-compression, Zod
**Storage**: Supabase PostgreSQL (recipes table) + Supabase Storage (recipe-images bucket)
**Testing**: Manual verification per quickstart scenarios + npm run lint
**Target Platform**: Mobile-first web (375px baseline), iOS Safari, Android Chrome
**Project Type**: Web application (Next.js)
**Performance Goals**: Upload <10s on mobile, page load <2s with images
**Constraints**: Images <500KB compressed (target ~300KB via client compression), WCAG AA contrast on overlays
**Scale/Scope**: Hundreds of recipes per user, ~1GB storage budget
